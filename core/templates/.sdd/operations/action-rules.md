# Action Rules

Map high-level user actions to the correct SDD workflow.

Global rules:
- Always work from an active spec when implementation is involved.
- Prefer safe non-code actions when ambiguous.
- Do not commit, push, create branches or write externally unless explicitly instructed.
- Do not implement from issue tracker or brief text alone.

Supported actions:
- prepare-spec
- create-spec-from-source
- split-source
- implement-next-task
- review-diff
- resume-work
- close-spec
- sync-external
